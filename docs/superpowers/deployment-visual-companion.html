Libreville Eats Deployment

Current proposed target: Frontend on Netlify, Backend on Render, Database on MongoDB Atlas.

Frontend

Next.js app hosted on Netlify

Build output served as static / serverless pages. Makes API calls to backend.

→

Backend

Node.js / Express API hosted on Render

Receives requests from frontend, validates, authenticates, accesses MongoDB, publishes/retrieves real-time socket events.

→

Database

MongoDB Atlas cluster

Stores users, restaurants, commandes, livreurs. Backend uses Mongoose + connection string from environment.

Additional deployment considerations

  • Environment variables: API URL, MongoDB URI, JWT_SECRET, Redis host/port if used.
  • Backend static asset or CORS config to allow frontend origin.
  • Production start script: `npm start` for backend, `next build && next start` or Netlify build command for frontend.
